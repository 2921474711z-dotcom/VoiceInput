# 配置模板页重构设计

目标：把 `/config` 页里的“配置模板”子页改成职责清晰的模板管理界面，解决“新增、编辑、列表、详情揉在一起”的问题。

## 问题

- 新增模板区、模板配置明细、模板列表详情处于同一视觉层级。
- 用户看不出当前主操作是“新建模板”还是“维护已有模板”。
- 模板列表默认展开大量配置项，信息噪音过高。
- “配置管理”与“配置模板”之间边界不清，容易误解为模板只是当前系统配置的别名。

## 设计

### 1. 页面分成上下两块

- 上半区：模板编辑器
  - 只负责“新建模板”或“修改模板”。
  - 包含模板名称、模板说明、是否设为默认模板、模板参数表单、保存/取消操作。
  - 明确提示“这里编辑的是模板自己的配置，不依赖配置管理当前值”。

- 下半区：模板列表
  - 只负责浏览和维护已有模板。
  - 每条模板默认只显示名称、说明、默认状态、时间、摘要。
  - 详情默认折叠，点击“查看详情”再展开完整配置项。

### 2. 操作路径清晰化

- `修改`：把当前模板载入上半区编辑器。
- `保存模板修改`：写回数据库。
- `取消修改`：退出编辑态，恢复到新建模板态。
- `查看详情`：仅展开该模板的只读配置详情。
- `设为默认模板`、`删除`、`载入到配置管理`：保留在列表项动作区。

### 3. 视觉层级

- 模板编辑器使用一块主卡片，强调“当前正在编辑什么”。
- 模板列表每项保持摘要式，不再一开始铺满两列配置块。
- 模板详情只在当前行内展开，避免整页信息重复。

## 不改范围

- 不改“配置管理”子页业务逻辑。
- 不改“使用统计”子页。
- 不改模板在工作台执行链路中的后端生效方式。
